Git Workflows for RDRP

Overview

This guide covers two essential Git workflows for collaborative development: working with branches on the same repository and contributing to projects via forking.

Workflow 1: Clone → Branch → Push → Pull Request (Same Repo)

Use this workflow when you have direct access to the repository (you're a collaborator).

  1. Clone the repository 
    git clone <repository-url>
cd <repository-name>
  2. Create and switch to a new branch 
    git checkout -b feature/your-feature-name
    Note: Use descriptive branch names like feature/add-login or fix/navbar-bug
  3. Make your changes and commit them 
    git add .
git commit -m "Add descriptive commit message"
  4. Push your branch to the remote repository 
    git push origin feature/your-feature-name
  5. Create a Pull Request
    • Go to the repository on GitHub
    • Click "Compare & pull request" button
    • Add a title and description for your changes
    • Click "Create pull request"

Workflow 2: Fork → Clone → Branch → Push → Pull Request (Fork Contribution)

Use this workflow when contributing to a project you don't have direct access to.

  1. Fork the repository
    • Go to the original repository on GitHub
    • Click the "Fork" button in the top-right corner
    • This creates a copy of the repo in your GitHub account
  2. Clone your forked repository 
    git clone <your-fork-url>
cd <repository-name>
  3. Add the original repository as upstream (optional but recommended) 
    git remote add upstream <original-repository-url>
  4. Create and switch to a new branch 
    git checkout -b feature/your-contribution
  5. Make your changes and commit them 
    git add .
git commit -m "Describe your changes clearly"
  6. Push the branch to your fork 
    git push origin feature/your-contribution
  7. Create a Pull Request to the original repository
    • Go to your forked repository on GitHub
    • Click "Compare & pull request" button
    • Ensure the base repository is the original repo and base branch is usually main
    • Ensure the head repository is your fork and compare branch is your feature branch
    • Add a clear title and description
    • Click "Create pull request"

Best Practices

  • Always work on feature branches - Never commit directly to main
  • Pull before you start - Make sure you have the latest changes
  • Write clear commit messages - Explain what and why, not just what
  • Keep pull requests focused - One feature or fix per PR
  • Test your changes - Make sure everything works before creating a PR
  • Review your own code - Check the diff before submitting

Keeping Your Fork Updated

When working with forks, periodically sync with the original repository:

git fetch upstream
git checkout main
git merge upstream/main
git push origin main

Common Commands Summary

Essential Commands

# Check current branch and status
git status
git branch

# Switch branches
git checkout <branch-name>

# Create and switch to new branch
git checkout -b <new-branch-name>

# Add and commit changes
git add .
git commit -m "Your message"

# Push to remote
git push origin <branch-name>

# Pull latest changes
git pull origin <branch-name>
